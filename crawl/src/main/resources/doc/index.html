Introduction

Normal usage of XML2OWL requires the user to provide XML data files. However, for many data sources, only HTML is available. Therefore, this component first extracts XML data files from HTML sources (with the help of the Webharvest library), which can then be used as input to XML2OWL. This greatly expands the scope of potential XML2OWL usage.

Installation

The web crawler component is installed with Maven. From the crawl directory, run "mvn install", which includes a demo run. To skip the demo, run "mvn install -DskipTests=true" instead.

Usage

The Java program expects seven arguments:

  • commonPath: a directory relative to which the other files are located
  • webharvestFile: the Webharvest configuration file
  • downloadDirectory: the directory to download HTML sources to
  • owlFile: the existing OWL ontology file to be enriched
  • rulesFile: the XML2OWL XML rules file prepared by the user
  • harvest: a flag indicating whether to do a fresh download first
  • map: a flag indicating whether to run XML2OWL on the generated XML datafile

For example, in order to run the included bookdepository test manually without immediately running XML2OWL on the result, run:

% java -jar target/xml2owl-crawl-1.0.0-jar-with-dependencies.jar src/test/resources/data/books bookdepository/webharvest.xml bookdepository/downloads owl.xml bookdepository/rules.xml true false
